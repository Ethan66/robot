    # git && npm

    几个不常用却很实用的Git小命令 (opens new window) git tag创建、远程推送、回退以及强推push -f (opens new window)

    # git

    # git pull, git fetch, git pull --rebase区别

    • git fetch：只是拉取代码，不会合并代码
    • git pull：等价于git fetch + git merge，所以会产生类似Merge branch 'xxx' into 'xxx'提交信息
    • git pull --rebase：表示拉取代码的时候，自动变基，但是自动变基需要工作区要干净
    • rebase遇到冲突：解决冲突，然后git add .，再git rebase --continue。想要回到git pull之前的状态，只需要退出rebase模式：git rebase --abort

    # git 自动变基命令行

    两条命令让你的git轻松自动变基，学到了！ (opens new window)

    $ git config --global pull.rebase true // 每次git pull的时候自动转为git pull --rebase，自动变基需要工作区要干净，所以需要第二个命令

$ git config --global rebase.autoStash true // 每次自动变基会将工作区的内容自动stash，rebase完成自动恢复

    # git 问题排查

    // 场景：找到历史提交记录那一条报错了
$ git bisect start HEAD 4d83cf // HEAD表示最近一次提交，比如第101次提交(是终点)，4d83cf表示第一次提交(是起点)。此时代码库会切换到中点，第51次提交记录
$ git bisect good // 运行完代码逻辑，假如没有问题执行此命令，表示第51次代码没有问题，有问题的是在52-101次提交记录，代码库会自动切换到中点，第76次提交记录
$ git bisect bad // 运行完代码逻辑，假如有问题执行此命令，表示第76次代码是有问题的，所以范围变为52-76次代码，代码库自动切换到中点，第63次提交记录
...直到找到错误提交的那次记录为止
$ git bisect reset // 退出查错，回到最近一次的提交记录

    # 增删分支

    // 新建分支
$ git checkout --orphan [name] // 创建没有任何历史记录的分支
$ git branch [name] // 新建停留在当前分支
$ git checkout -b [name] // 新建并切换到新分支
$ git checkout [name] // 切换分支

// 分支连接远程分支

// 远程分支不存在，先将本地分支推送到远程
$ git push -u origin master // 将本地master分支推送到远程

// 远程分支存在
$ git checkout -b [name] origin/master // 新建并切换新分支，并将新分支和远程master分支关联：（注：新分支的代码是远程master的代码，未commit的代码会一起带过去的）

// 删除分支
$ git branch -D [name]
$ git push origin --delete [name] // 删除远程分支

// 切断本地分支和远程分支的关联
git branch --unset-upstream

    # git reset

    // 其中HEAD代表版本库，index代表暂存区

$ git reset(等价于git reset --mixed)：
场景：不想提交本次修改和本次添加的文件
回退版本库，暂存区。会保留工作区代码。只是将git commit和index 信息回退到了某个版本.

$ git reset --soft：
场景：不想提交本次修改，但是添加的文件还是要提交的
回退版本库。保留暂存区和工作区源码。只回退到commit 信息

$ git reset --hard：
$ git push -f
$ git reset --hard HEAD~1 // 回到上一个commit
场景：多用于远程回滚
回退版本库，暂存区和工作区。源码也会回退到某个版本,commit和index 都回回退到某个版本

$ git revert:
git revert到某个版本
git push 到远程，2种不同的是git revert 会产生新的commit，你还可以回退

    # git tag

    // 场景：demo项目，线上代码回滚版本

// 打标签：
$ git tag v1.0 // 默认是给最近一次的commit打标签
$ git tag v0.9 fd23001 // 给历史commit记录打标签

// 打标签并添加备注
$ git tag -a v0.1 -m '我是备注内容'
$ 

// 查看标签
$ git tag
$ git show v0.1 // 查看单个标签（可以看到备注内容）

// 标签推送到远程服务器
$ git push origin [tagName] // 推送单个标签
$ git push origin --tags // 推送所有的标签

// 标签回滚代码
$ git reset --hard v1.0 // 不需要记录commit记录，方便回滚

// 删除标签
$ git tag -d v1.0

// 删除远程标签
$ git tag -d v1.0
$ git push origin :refs/tags/v1.0
