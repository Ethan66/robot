    # DOM

    # DOM

    1、Document Object Model: 文档对象模型。DOM的目的就是为了能让js操作html元素而制定的一个规范。 2、DOM就是由节点组成的。包括：元素节点。文本节点：标签中的文字（比如标签之间的空格、换行）。属性节点：标签的属性。顶级是html节点 3、HTML加载完毕，渲染引擎会在内存中把HTML文档，生成一个DOM树，getElementById是获取内中DOM上的元素节点。然后操作的时候修改的是该元素的属性。

    # 获取元素

    获取body、html和一般元素
    // 获取body和html节点
document.body
document.documentElement

// 获取元素
document.querySelector("body")
document.querySelector('div') // 假如有2个div取第一个
document.querySelectorAll("div")[0]

    # 获取父子元素

    获取父子，兄弟元素
    // 获取父元素
document.querySelector('body').parentNode
document.querySelector('body').parentElement

// 获取子节点
let el = document.querySelector('body')
el.firstElementChild // 获取第一个子元素
el.lastElementChild // 获取最后一个子元素
el.firstChild // 获取第一个子节点，不实用
el.lastChild // 获取最后一个子节点，不实用

// 获取兄弟节点
let el = document.querySelector('div')
el.nextElementSibling // 获取下一个元素节点
el.previousElementSibling // 获取上一个元素节点
el.nextSibling // 获取下一个节点，不实用，因为换行也是一个节点
el.previousSibling // 获取上一个节点，不实用，因为换行也是一个节点

// 获取所有兄弟元素
let el = document.querySelector('div')
el.parentNode.children // 获取所有子元素
el.parentNode.childNodes // 获取所有子节点

    # 事件

    # 四种事件模型的区别

    // DOM0事件模型：直接给元素绑定事件属性
<div onclick="fn()">btn</div>
element.onclick = function () {} // 执行的函数以最后一个绑定的函数为准

// 删除事件模型
element.onclick = null

// DOM2事件模型
// IE通过attachEvent绑定：有2个过程：事件处理阶段和事件冒泡阶段，先执行目标元素绑定的事件，再从目标元素冒泡到document，最后到window

通过addEventListener绑定，有3个过程：事件捕获阶段，事件处理阶段和事件冒泡阶段。先从window到document一直向下传播到目标元素，依次检查经过的节点是否绑定了事件监听函数，如果有则执行。后面两个阶段和 IE 事件模型的两个阶段相同
 element.addEventListener('click', function () {}, false);

// 1、true表示捕获阶段触发，false表示冒泡阶段触发(默认)
// 2、一般的浏览器：div -> body -> html -> document -> window
// 3、绑定了2个函数，都会执行
// 4、不能冒泡的事件: blur、focus、load、unload、onmouseenter、onmouseleave

 // DOM3事件模型：增加很多事件类型，如焦点事件，鼠标事件，键盘事件
 element.addEventListener('keyup', function () {}, false);

    # 事件流

    • 捕获阶段–事件从 window 开始，然后向下到每个元素，直到到达目标元素事件或event.target。
    • 目标阶段–事件已达到目标元素。
    • 冒泡阶段–事件从目标元素冒泡，然后上升到每个元素，直到到达 window。

    // 点击事件：
onclick, ondblclick // 双击

// 键盘事件：
onkeyup // 释放键盘时, onkeydown // 按下键盘时

// 焦点事件：
oninput // 文本框输入时, onchange // 文本内容改变时, onfocus // 文本框聚焦时, onblur // 失焦时

// 鼠标事件
onmouseover // 鼠标移进事件源时, onmousemove // 鼠标在事件源移动时, onmouseout // 鼠标移出时

    # 事件对象

    1、触发DOM上的某个事件时，都会产生一个事件对象event。 2、兼容性：event = event || window.event(IE6,7,8)

    // event常用属性
event.bubbles // true表示事件可以冒泡，不是所有的事件都可以冒泡
event.stopPropagation() // 阻止冒泡，当bubbles为true可以使用
event.cancelable // 为true表示可以取消事件的默认事件
event.preventDefault() // 取消默认事件，当cancelable为true时
event.currentTarget // 执行事件时的那个元素，即currentTarget始终 === this
event.targget // 真正触发的事件的元素,比如事件点击的元素
event.type // 被触发的事件的类型,如'click','input'

    # 样式

    # 记忆方法

    • clientWidth/clientHeight: 获取content+padding
    • clientTop/clientLeft: 获取border
    • offsetWidth/offsetHeight: 获取content+padding+border
    • offsetTop/offsetLeft: 元素没有设置left, top时获取的margin
    • getComputedStyle(el).margin: 获取元素的margin（带px）
    • getComputedStyle(el).marginTop: 获取元素的marginTop（带px）

    # 获取元素的各种宽高度

    let el = document.querySelector('.cls1')
// 元素的所有宽高（content、padding、border）
el.offsetWidth
el.offsetHeight

// 元素的宽高（content和padding）
el.clientWidth
el.clientHeight

// 元素的border值
el.clientTop // 元素的上边框宽度
el.clientLeft // 元素的左边框宽度

// 元素的margin值
getComputedStyle(el).marginTop // 注意：展示的是数值带px
getComputedStyle(el).margin // 注意：展示的是数值带px，比如：'5px 6px 7px 8px'

    # 鼠标的偏移量

    • clientX：当鼠标事件发生时，鼠标相对于浏览器（这里说的是浏览器的有效区域）x轴的位置；
    • clientY：当鼠标事件发生时，鼠标相对于浏览器（这里说的是浏览器的有效区域）y轴的位置；
    • screenX：当鼠标事件发生时，鼠标相对于显示器屏幕x轴的位置；// 不常用
    • screenY：当鼠标事件发生时，鼠标相对于显示器屏幕y轴的位置；// 不常用
    • offsetX：当鼠标事件发生时，鼠标相对于事件源x轴的位置
    • offsetY：当鼠标事件发生时，鼠标相对于事件源y轴的位置
