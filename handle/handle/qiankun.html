    # qiankun

    # 前置知识

    vue-cli的环境变量：1. vue.config.js中的publicPath配置好后，process.env.BASE_URL就可以在代码中使用了；2. 可以设置.env文件，来生成变量，给vue.config.js使用；3. 线上和本地取不同的值，可以创建.env.development和.env.production文件，在文件里设置变量VUE_APP_XX_XX=//localhost:7777/sub-vue/来实现

    vite的环境变量：也是在.env.development和.env.production文件中设置，VITE_XX_XX=//localhost:7777/sub-vue/，使用：import.meta.env.VITE_XX_XX

    # 主逻辑

    1. 主应用：相当于一个容器，用来注册微应用和监听url，当微应用注册后，一旦url 发生变化，便会自动触发 qiankun 的匹配逻辑，所有 activeRule 规则匹配上的微应用就会被插入到指定的 container 中，同时依次调用微应用暴露出的生命周期钩子。
    2. 微应用：业务项目，不需要安装任何依赖，新增内容：1.新增public-path.js文件，用来改变webpack中所有静态资源的加载路径。2. entry.js：引入public-path.js，将dom render进行单独封装为render函数，export bootstrap、mount、unmount三个方法，将render方法放入mount中。3. 改变build方式，把子应用打包成 umd 库格式，能使主应用访问子应用的bootstrap、mount、unmount方法。

    # 步骤（vue2 + vue-cli）

    1. 新增public-path.js文件，用来改变webpack中所有静态资源的加载路径。
    2. entry.js：引入public-path.js，将dom render进行单独封装为render函数，export bootstrap、mount、unmount三个方法，将render方法放入mount中。
    3. 改变build方式，把子应用打包成 umd 库格式，使主应用访问子应用的bootstrap、mount、unmount方法。

    # 改变webpack中所有静态资源的加载路径

    // public-path.js
(function () {
    if (window.__POWERED_BY_QIANKUN__) {
      if (process.env.NODE_ENV === 'development') {
        __webpack_public_path__ = `//localhost:${process.env.VUE_APP_PORT}${process.env.BASE_URL}/`
        // __webpack_public_path__ = `//localhost:7000/aabb/`
        return
      }
      __webpack_public_path__ = window.__INJECTED_PUBLIC_PATH_BY_QIANKUN__
      // __webpack_public_path__ = `${process.env.BASE_URL}/`
    }
  })()

    # 改变main.js

    // main.js
import './public-path'
import Vue from 'vue'
import App from './App.vue'
import router from './router'

let vm = null

function render(props = {}) {
  const { container } = props
  vm = new Vue({
    router,
    render: h => h(App)
  }).$mount(container ? container.querySelector('#app') : '#app')
}

// 独立运行时
if (!window.__POWERED_BY_QIANKUN__) {
  render();
}
// 初始化执行（只有一次—）
export async function bootstrap() {
  console.log('[vue] vue app bootstraped');
}
// 挂载执行（每次进入应用时都会执行）
export async function mount(props) {
  console.log('[vue] props from main framework', props);
  render(props);
}
// 卸载执行（每次离开应用时都会执行）
export async function unmount() {
  vm.$destroy();
  vm.$el.innerHTML = '';
  vm = null;
}

    # 问题

    • 问题： 本地运行主应用的地址是：http://localhost:8080/，但是本地调试运行的地址是：http://localhost:3000/unicom/，导致http://localhost:3000/unicom/aa.js等资源加载失败
    • 解决方法：vite.config.js中配置base属性：base: mode === 'development' ? '/' : '/unicom/'，将所有静态资源前面加载/unicom/统一前缀，这样就能访问静态资源了，也可以让本地调试还是http://localhost:8080/
