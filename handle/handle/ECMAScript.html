    # ECMAScript

    # 字符串中的&nbsp;&nbsp;

    假如用含有&nbsp;的字符串放入html中，显示的是&nbsp;，不是空格，需要换成\xa0，那这是为什么呢？

    1. &nbsp; 是 HTML 实体，用于在 HTML， 中表示不间断空格。
    2. \xa0 是 Unicode 字符编码的十六进制表示形式，用于在 JavaScript 字符串中表示不间断空格。

    # ES2022(ES13)

    # 正则匹配索引(开始和结束)

    以前的正则只是返回匹配的开始索引。有些场景需要结束索引。这时候可以添加特殊的标志[d]，只对exec和match的api有用，并存放在indices 对象中

    var str = '1231sdfsd34543'
const arr = str.match(/\d+/d)
console.log(arr.indices[0]) // [0, 4]

    # 空值合并运算符(??，和||对应)

    ||：左侧的值是0 '' false null undefined NaN，返回右侧值，否则返回左侧值； ??：左侧的值为 null 或者 undefined 时，返回其右侧值，否则返回左侧值

    if((value ?? '') !== ''){} // 等价：if(value !== null && value !== undefined && value !== ''){}
console.log(undefined ?? 'co') // 'co'

    # 逻辑或分配(||=)

    左侧表达式为 falsy 值（虚值） 时才赋值

    // 等价于x || (x = y)
const a = { time: 50, unit: '' }
a.time ||= 10 // time: 50
a.speed ||= 20 // speed: undefined
a.unit ||= 20 // unit: 20

    # ES2014(ES5)

    # 逗号操作符

    对每个操作对象求值(从左到右)，返回最后一个操作对象的值

    var x = 0
var b = (x++, x) // b === 1
const result = arr => (arr.push('a'), arr) // 不能直接拿push，所以需要逗号操作符

    # JSON.stringify

    JSON.stringify在控制台打印不好看，可以用第二参数，第三参数进行简化

    // 1. 对象属性太多，想看到个别对象，传第二个数组参数（只适合平级对象）
const obj = { a: 1, b: 2, c: 3, d: 4 }
console.log(JSON.stringify(obj, ['a', 'b']))

// 2. warning: 多层级对象不适合传第二个数组参数
const obj1 = { a: { b: 1 }, c: 2, d: 3 }
console.log(JSON.stringify(obj1, ['a', 'c']))

// 2. 对象属性太多，且要清楚层级
const obj2 = { a: { b: 1 }, c: 2, d: 3 }
console.log(JSON.stringify(obj2, null, 2))
