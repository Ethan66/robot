    # 初始化

    白话文 (opens new window) 源码版 (opens new window)

    • 代码: vue-learn-code/Vue2.5.16/computed.html。

    • 初始化Vue实例时，在beforeCreate和created之间，会执行initState()函数，这个函数包含处理props，data，computed，watch等数据。处理$options.computed，执行initComputed(vm, vm.$options.computed)。

    • 创建空对象wwathcers = vm._computedWatchers，遍历computed每一个属性key，把每个属性的属性值value作为get方法（value是对象直接获取value.get），存给getter变量，watchers[key] = new Watcher(vm, getter, lazy = true)，lazy表示在脏值，脏值为true表示需要计算，不能从缓存中拿。此时还没有跳出initComputed方法。

    • new Watcher(vm, getter, lazy = true)，和watch配置的时候一样（实例作为this，this.vm = vm， vm._watchers.push(this)，this.deep = options.deep， this.id = id），不一样的是：this.cb = 空函数，this.dirty = lazy（true），this.getter = getter，this.value = this.lazy ? undefined : this.get()，所以一开始this.value === undefined，也不会执行getter方法，跳出Watcher实例。

    • 所以vm._watchers里存有watch和computed的watcher，vm._computedWatchers只存computed的watcher

    • 判断vm是否存在key属性，不存在调用defineComputed(vm, key, getter)。给vm[key]进行响应式。

    • get方法：判断vm._computedWwatchers里是否有key属性的value值，有判断watcher.dirty是否等于true，判断从缓存获取还是执行方法获取。存在Dep.target，执行watcher.depend()，将Dep.target存入watcher.dep.subs中。

    • set方法：vm.$options有配置就传，没有就设置为空函数。

    # initComputed(vm, vm.$options.computed)

    initComputed(vm, vm.$options.computed)
    function initComputed(vm, computed) {
  var watchers = vm._computedWatchers = {}
  for(var key in computed) {
    var getter = typeof comoputed[key] === 'function' ? computed[key] : computed[key].get
    watchers[key] = new Watcher(vm, getter, function(){}, { lazy: true })
    if (!(key in vm)) {
      defineComputed(vm, key, computed[key])
    }
  }
  }

  // 重要函数
  var Watcher = function (vm, expOrFn, cb, options) {
    this.vm = vm
    vm._watchers.push(this)
    if (options) {
      this.deep = !!options.deep
      this.lazy = options.lazy // 脏值
    } else {
      this.deep = this.lazy = false
    }
    this.cb = cb //回调函数
    this.id = Math.random()
    this.newDeps = []
    if (typeof expOrFn == 'function') { // watcher是字符串
      this.getter = expOrFn
    }
    this.value = this.lazy ? undefined : this.get()
  }
      
      
  // 辅助函数
  Watcher.prototype.get = function () {
    pushTarget(this) // 给Dep添加target
    var value = this.getter.call(this.vm, this.vm)
    if (this.deep) {
      traverse(value) // 变量每一个属性进行监听
    }
    popTarget() // 删除Dep.target
    return value
  }

  // 给vm[key]添加响应式
  defineComputed (target, key, userDef) {
    var sharedPropertyDefinition = { //共享属性定义
      enumerable: true,
      configurable: true,
      get: function(){},
      set: function(){}
      }
    if (typeof userDef === 'function') {
      sharedPropertyDefinition.get = createComputedGetter(key)
      sharedPropertyDefinition.set = function () {}
    } else {
      sharedPropertyDefinition.get = createComputedGetter(key)
      sharedPropertyDefinition.set = userDef.set()
    }
    Object.defineProperty(target, key, sharedPropertyDefinition)
  }

  function createComputedGetter(key) {
    return function computedGetter () {
      var watcher = this._computedWatchers[key]
      if (watcher) {
        if (watcher.dirty) {
          watcher.value = watcher.get()
          watcher.dirty = false
        }
        if (Dep.target) {
          watcher.depend()
        }
        return weatcher.value
      }
    }
  }

    # computed里的data数据B改变

    当B改变时，会通知B闭包中的dep中所有的依赖subs，即watcher.update()被触发，update()只是将watcher.dirty= true。当再次读取computed里的数据时，就会触发新get()，进行更新。

    # 页面P引用了computed数据C，C引用了data数据D，D修改，变化顺序是怎么样的

    其实D会同时收集C和P的watcher，当D修改后，会遍历顺序通知wather，先通知C的watcher，将this.dirty = true，此时还没有执行C的wather.getter(),值还没有更新。所以再去通知P的watcher，P的watcher执行后，重新渲染页面，需要重新获取C的值，此时因为C.watcher.dirty===true, 所以不会拿缓存值，调用C.watcher.getter() 重复计算。
