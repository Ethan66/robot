    vue从创建到挂载的过程

    掏钱的机器猫 2020/10/23 vue2源码

    # vue从创建到挂载的过程

    # 初始化vue实例

    参考 (opens new window)

    • 代码: vue-learn-code/Vue2.5.16/lifecycle.html。
    • 执行new Vue(options)，执行vue._init(options)。

    # beforeCreate之前

    初始化Vue实例时，执行vue._init()方法，调用mergeOptions()方法，校验配置的options的规范，如果配置项里有mixins，递归调用mergeOptions方法，把mixins的配置项匹配给parent，再遍历配置的options，把每项与父级构造函数的options属性进行合并（如果childVal不存在，就返回parentVal,如果存在，就把childVal添加到parentVal后，比如生命钩子，所以mixins里的生命周期早于当前实例执行），将得到一个新的options选项赋值给$options属性。再执行beforeCreate钩子，此时还没有$el和$data。 所以beforeCreate之前是将options配置项检验合并。

    # vue._init()

    • _init方法初始化一些属性，vm._uid（唯一id）, vm._self = vm, vm._isVue = true。
    • 调用mergeOptions()，返回值给$options。
    • 调用initLifecycle(), 初始化vm.$parent, vm.$root, vm.$children, vm.$refs, vm._watcher等
    • 调用callHook(vm, 'beforeCreate'): 因为mergeOptions已经将parent.beforeCreate和options.beforeCreate都合成一个数组$options.beforeCreate了，所以遍历调用就行了

    # mergeOptions()

    • mergeOptions(resolveConstructorOptions(vm.constructor), options || {}, vm)
    • 参数：resolveConstructorOptions(vm.constructor): 解析constructor上的options属性，一般为{}，当做parent
    • 参数：options: 配置options
    • 校验options规范：校验子组件components的name规范，props是否是对象数组等
    • 合并mixins: 执行mergeOptions(parent, child.mixins[i], vm)，返回值传给parent
    • 先遍历parent的属性key，执行mergeField(key)
    • 后遍历options的属性key，将parent不存在的属性key进行mergeField(key)

    # mergeField(key)

    • strats变量：策略者模式，每个key存着相对应的方法，比如strats.el()，strats.data()。取出来赋值给strat变量
    • 执行方法strat()，将parent的key属性的value和options的key属性的value进行合并，返回的值赋值给新的options的属性key
    • 最后return options，跳出mergeOptions()方法
    strats.data()：childVal为options.data, parentVal为parent.data
    function () {
  return function mergedInstanceDataFn() {
    // instance merge
    var instanceData = typeof childVal === 'function'
        ? childVal.call(vm, vm)
        : childVal;

    var defaultData = typeof parentVal === 'function'
        ? parentVal.call(vm, vm)
        : parentVal;

    if (instanceData) {
        return mergeData(instanceData, defaultData)
    } else {
        return defaultData
    }
  }
}

    # created之前

    调用initState()方法，处理$options里每个属性，先处理props，再给data实现数据响应，最后给computed，watch添加观察者。再执行created钩子。

    # initInjections(vm)

    初始化provide和inject配置

    # initState(vm)

    • initProps(vm, opts.props): 判断是否有$options.props，添加观察者模式
    • initMethods(vm, opts.methods): 判断是否有$options.methods, 初始化事件
    • initData(vm)
    • initComputed(vm, opts.computed): 判断是否有$options.computed，初始化计算属性
    • initWatch(vm, opts.watch): 判断是否有$options.watch，进行初始化watch

    # initData(vm)

    • 判断$options.data是方法还是对象，对象直接给vm._data，否则执行$options.data.call(vm, vm), 返回给vm._data
    • 判断vm_data和$options.props和$options.methods属性是否重复，重复报警告
    • 遍历每个属性执行proxy(vm, '_data', key)，进行数据代理：vm.key指向vm._data.key
    • 对vm._data的每一个属性进行响应化

    # beforeMounted之前

    1. vue有2个版本的代码：runtime only（只包含运行时版本）和runtime+compiler（同时包含编译器和运行时的完整代码）。他们的区别就是编译器，将template转化为render函数。实际开发用runtime only，同时借助webpack的vue-loader，通过这工具来将模板编译成render函数。
    2. 所以两个版本区别在于：vue.$mount()方法是否将vue.prototype.$mount方法缓存起来，直接执行$mount的就是runtime only版本，否则就是完整版本。完整版先根据传入的el获取dom元素，判断有无配置render函数，没有的话判断配置是否有template模板，是字符串，或者是dom的直接获取innerHtml，没有配置template就是el作为innerHtml。将模板字符串编译成render函数。传给$options.render，再执行原始vue.prototype.$mount()函数。

    # vm.$mount(vm.$options.el)

    vm.$mount为手动挂载函数

    • query(el): 获取dom元素
    • 判断$ptions.render是否存在
    • 不存在$options.render，判断$options.template类型，是字符串（dom的id）或者dom元素，获取innerHTML；没有$options.template，获取$options.el外层的dom的innerHTML，获取template模板字符串
    • 不存在$options.render，将template模板字符串通过compileToFunctions()方法转为render函数，将转化后的render函数赋值给$options.render
    • 执行原先绑定在vue.prototype.$mount的方法，mount.call(this, el)，方法里有callHook(vm, 'beforeMount')，此时还没有跳出mount方法

    # callHook(vm, 'beforeMount')

    # mounted之前

    • 开始挂载前，创建new Watcher(vm, updateComponent)实例watcher，updateComponent为外部定义的方法，将它存在watcher.update中。
    • 执行updateComponent()，方法里先执行vm._render()，获取最新的Vnode节点树，同时获取了vm.data属性，vm.data的属性监听了watcher。此时还没跳出函数，再执行vm._update()方法，此时没有旧的Vnode节点树，直接更新DOM，完成第一次渲染。
    • 执行callHook(vm, 'mounted')

    # beforeUpdate

    修改数据，触发watcher.update方法，判断vm._isMounted === true（已被挂载过了），执行callHook(vm, 'beforeUpdate')，再执行updateComponent()，将最新的Vnode节点树和上一次渲染的Vnode节点树对比，并更新DOM节点（即patch操作），再执行callHook(vm, 'updated')

    # beforeDestroy

    • 执行vm.$destroy方法，执行callHook(vm, 'beforeDestroy')，没有跳出函数。
    • 判断当前实例有父级实例，并且父级实例没有被销毁且不是抽象组件，那么将当前实例从父级实例的$children属性中删除。
    • 移除依赖：遍历vm._watchers，调用每一个watcher的teardown方法。
    • 移除响应式数据的引用：vm._data.ob.vmCount--
    • 移除Vnode: vm.patch(vm._vnode, null)
    • callHook(vm, 'destroyed')
